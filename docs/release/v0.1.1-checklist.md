# v0.1.1 Release Checklist

Use this checklist to ship `v0.1.1` with predictable quality.

## 1) Scope Freeze

- [ ] All targeted issues are linked to the release milestone.
- [ ] No new features added after freeze without explicit approval.
- [ ] Changelog entries prepared under `Unreleased`.

## 2) Code Health

- [ ] `flutter analyze` passes.
- [ ] `flutter test` passes.
- [ ] Proxy server smoke test passes (`npm start` + endpoint checks).
- [ ] No debug prints or TODOs in release-critical code paths.

## 3) Product Quality

- [ ] Dashboard loads gracefully on slow network.
- [ ] News and company screens handle empty/error states.
- [ ] Search UX returns stable results for partial queries.
- [ ] Retry actions exist for recoverable network failures.

## 4) Security & Config

- [ ] `.env` is not committed.
- [ ] `.env.example` reflects all required keys.
- [ ] API keys rotated if previously exposed.
- [ ] Release builds do not depend on debug signing.

## 5) Release Artifacts

- [ ] Version bumped in `pubspec.yaml` (e.g. `0.1.1+2`).
- [ ] Tag created (`v0.1.1`).
- [ ] GitHub release notes drafted from `CHANGELOG.md`.
- [ ] Store-specific release notes prepared (iOS + Android).

## 6) Post-Release

- [ ] Monitor crash/error logs after release.
- [ ] Track key regressions in a hotfix board.
- [ ] Start `v0.1.2` planning with recorded learnings.
